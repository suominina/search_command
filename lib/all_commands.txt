Linuxコマンド一覧

env  //環境変数を表示
grep [オプション] [文字列パターン] [検索対象ファイル] //テキストファイルの内容を検索し、指定した正規表現にマッチする行があれば表示
	-F 文字列パターンを正規表現ではなく単なる文字列として扱う
	-c マッチした行数だけ表示
	-i 大文字と小文字を区別しない
	-lマッチした行のあるファイル名のみを表示
	-n 行番号も合わせて表示
	-v マッチした行だけでなくマッチしなかった行も表示
pgrep [オプション] [パターン] //現在実行中のプロセスを調べて、ユーザ名、UID、GIDなどをもとにプロセスIDを検索
	-t 端末 // 指定した端末で実行されているプロセスIDを検索
	-u euid //指定された実効ユーザ(プログラムが動作するときの権限)IDに該当するプロセスIDを検索
	-U uid //指定された実ユーザ(プログラムを起動したユーザ)IDに該当するプロセスIDを検索
fgrep [検索文字列] [ファイル] //正規表現を使用できないgrep

export $変数 //環境変数を設定
set [オプション] //オプションを指定しないとシェル変数、シェル関数を一覧表示
	-o シェルオプション //シェルオプションを有効にする
	+o シェルオプション //シェルオプションを無効にする
	noclobber か -C //リダイレクションによる既存ファイルへの上書き禁止する
	histexpand か -H //「!番号」による履歴参照を行う
man [オプション] [セクション] コマンドまたはキーワード /*引数で指定したコマンドのオンラインマニュアルを表示。ディストリビューションに含まれるパッケージをインストールした場合は、/user/share/manの下の街頭するセクションのサブディレクトリに、含まれないソフトウェアをインストールした場合/user/local/share/manが標準*/
	-f //キーワードと完全に一致するマニュアルが何セクションにあるか表示
	-k //キーワードを含むマニュアルが何セクションにあるか表示
history [オプション] [履歴数] //引数を指定しない場合、すべての履歴を表示、履歴数を指定すると最新から指定数の履歴を消去
touch ファイル名 //空のファイルを作成
cp ファイル1 ファイル2 //ファイル1をファイル2という名前でコピー
cp ファイル フォルダのパス //ファイルをフォルダにコピー
cp -r フォルダ1 フォルダ2のパス //フォルダ1とその中身を丸ごとフォルダ2にコピー
mkdir [オプション] ディレクトリ名
	-p //中間ディレクトリも同時に作成
	-m [アクセス権] //明示的にアクセス権を指定しディレクトリ作成
cat [オプション] [ファイル名] //ファイルの内容を表示、引数なしで実行するとキーボードからデータを読み取る
	-n //行番号を表示
	-b 空白行を飛ばして行番号を表示
	-A非表示文字を表示
	-s連続した空白行を1行に
	-E各行の最後に$をつける
tar [オプション] [ファイルかディレクトリ名]
	-c //アーカイブファイルを作成
	-t //アーカイブファイルの内容を表示
	-x //アーカイブを展開
	-f //アーカイブファイル名を指定
	-v //詳細情報を表示
	-i //アーカイブに対しbzip2を使用(圧縮、解凍)
	-j //アーカイブに対しxzを使用(圧縮、解凍)
	-z //アーカイブに対しgzipを使用(圧縮、解凍)
file [オプション] ファイル名かディレクトリ名 //ファイルの種類を調べる
	-i //MIMEタイプで表示
$ コマンド > ファイル //リダイレクト
	ls > file //lsの結果をfileに出力
$ コマンド >> ファイル//リダイレクト
	ls >> file //lsの結果をfileに追記
	ls 存在するファイル 存在しないファイル 2> file //エラーが出た時のみfileに出力
	ls 存在するファイル 存在しないファイル &> file //標準出力、標準エラー出力の両方をbothファイルに格納
	コマンド1 < file1 | コマンド2 //file1の内容を標準入力からコマンド1に取り込み、コマンド1の標準出力をコマンド２の標準入力に渡す
$ コマンド << 文字列 //指定した文字列が入力されるまで入力受付
$ コマンド1 ; コマンド2 //コマンド1の終了ステータスに関わらず、コマンド2を実行
$ コマンド1 && コマンド2 // コマンド1が実行できれば、コマンド2を実行
& コマンド1 || コマンド2 // コマンド1が実行できなければ、コマンド2を実行
head [オプション] [ファイル名] //先頭から任意に行数を指定し、内容を表示する
tail [オプション] [ファイル名] //末尾から任意の行数を指定し、内容を表示する
	-f //ファイルの内容が増え続けているものと仮定し、常にファイルの最終部分を読み続けようとする
tr [オプション] [文字群1[文字群2]] // 標準入力(キーボード)から入力した文字を指定した文字に変換し標準出力に出力
	-d //文字群1で合致した文字を削除する
	-s //文字群1で合致した文字の繰り返しを1文字に置き換える
sed [オプション] [編集コマンド] [ファイル名] //ファイルかパイプからの入力に対してテキスト変換
	-i //編集結果を直接ファイルに書き込む
	-e //条件を複数指定できる
	s/パターン/置換文字列/ //各行を対象に、最初にパターンに合致する文字列を置換
	s/パターン/置換文字列/g //ファイル内全体を対象に、パターンに合致する文字列を置換
cut [オプション] [ファイル名] //ファイル内の行中の特定部分のみを取り出す
	-c //指定された位置の各文字だけ表示
	-b //指定された文字の各バイトだけ表示
	-d //-fと一緒に用い、フィールドの区切り文字を指定。デフォルトはタブ
	-f //指定された各フィールドだけ表示
	-s //-fと一緒に用い、フィールドの区切り文字を含まない行を表示しない
wc [オプション] [ファイル名] //ファイル内の行数、単語数、バイト数を表示
	-c //バイト数だけ出力
	-l //行数だけ出力
	-w //単語数だけ出力
od [オプション] [ファイル名] //ファイルの内容を各進数で表示
	-d //10進数
	-x //16進数
	-c //ASCII文字またはバックスラッシュ付きのエスケープ文字として表示
	-A //表示されるオフセットの基数を選択するd, o, x, nで10, 8, 16, 表示なし
split [オプション] [入力元ファイル名[プレフィックス]] //ファイルを決まった大きさに分割。デフォルトで1000行ずつ
	-l 出力の行数を指定
	-d サフィックスを英字ではなく数字にする
nice [オプション] [コマンド] //プロセスの優先度を変更(-20~10), オプションなしだと10
	-n 数字 //入力された数字を優先度にする
renice [オプション] //実行中のプロセスの優先度を変更
	-p PID //PIDを指定する
jobs //バックグラウンドジョブと一時停止中のジョブを表示
bg % ジョブID //指定したジョブをバックグラウンドに移行
fg % ジョブID //指定したジョブをフォアグラウンドに移行
sleep 秒数 //指定した秒数コマンドの実行を遅らせる
	while true;do
    		date; sleep 2;
	done
kill [オプション] [シグナル名 | シグナル番号] プロセスIDもしくは%ジョブID //プロセスにシグナルを送る デフォルトで15
killall [オプション] [シグナル名 | シグナル番号] プロセス名 //複数起動している同じプログラムをまとめて終了

bc [オプション] //数値計算をするコマンド, 詳細はhttps://webkaru.net/linux/bc-command/
fdisk [オプション] デバイスファイル名 //パーティションの作成、削除、変更、情報の表示が可能
	-l //作成済みのパーティションテーブルを表示
fdisk デバイスファイル名 //対話形式で色々できる。サブコマンドで指定
	d //領域を削除
	l //パーティションのタイプを表示
	m //利用可能なメニューの表示
	n //新規パーティションの作成
	p //パーティションテーブルの表示
	q //変更を保存せず終了
	t //パーティションのタイプを変更
	w //テーブルをディスクに書き込み、終了
pvcreate [オプション名] [デバイス,パーティション名] //LVMの物理ボリュームを作成
	//pvcreate /dev/sda2 で/dev/sda2に物理ボリュームを作成
mkswap [オプション] [デバイスファイル名, ファイル名] //デバイス乗またはファイル乗に確保されたスワップ領域の初期化
	-C //不良ブロックのチェック
	-L ラベル名 //ラベルを指定し、そのラベルでswaponできるようにする
	-w0 //古い形式のスワップ領域を初期化する
	-v1 //新しい形式のスワップ領域を初期化する
swapon [オプション] [デバイスファイル名] //スワップ領域を有効にする
	-s //スワップの使用状況の表示
	-a ///etc/fstab中でswapマークがついているデバイスを全て有効にする
	-L ラベル名 //指定されたラベルのパーティションを有効にする
swapoff [オプション] [デバイスファイル名] //スワップ領域を無効にする
	-a /proc/swapsまたは/etc/fstab中のスワップデバイスやファイルのスワップ領域を無効にする
df [オプション] [デバイスファイル名, ディレクトリ名] //ファイルシステムの使用状況を表示
	-k //KB単位で表示する
	-h //容量に合わせた適切な単位で表示する
	-i inodeの使用状況をリスト表示する
du [オプション] [ディレクトリ名, ファイル名] //ファイル、ディレクトリの使用状況を表示
	-a //ディレクトリだけでなく、すべてのファイルについて容量を表示
	-h //容量に合わせた適切な単位で表示
	-s //指定されたファイル、ディレクトリの使用量の総和を表示
	-S //サブディレクトリの使用量を含めずに、ここのディレクトリの使用量を分けて表示する
tune2fs [オプション] デバイスファイル名 //ex2, ext3, ext4ファイルシステムのパラメータを適切な値に変更する
	-c 回数 //マウント数がこの回数を超えると、次回ブート次にファイルシステムをチェックする
	-i [d,m,w] //ファイルシステムをチェックする最大の時間間隔を設定する。数値のみもしくは末尾にdをつけると「日」,mは「月」wは「週」と解釈される
	-j //ext3ジャーナルをファイルシステムに追加する
	-l //ファイルシステムのフーパーブロックの内容を表示する
	-L ボリュームラベル //ボリュームラベルを設定
	-m 予約ブロックの比率 //予約ブロックの比率を%で指定する。デフォルトは5%
	-O 追加機能 //has_journalなど追加する機能を指定。先頭に^をつけると機能削除
dumpe2fs [オプション] デバイスファイル名 //ファイルシステムの詳細情報を表示
	-b //不良ブロックとして扱われているブロックを表示
	-h //スーパーブロックの情報だけを表示
fsck [オプション] [デバイスファイル名] //ファイルシステムの不整合をチェックし、必要であれば修正する。これを実行するときはファイルシステムをアンマウントする。そうしないと問題のないファイルを削除してしまう可能性がある。
	-t //チェックするファイルシステムのタイプを指定
	-s //fsckの動作を逐次的にする。複数のファイルシステムを対話的にチェックする際に使用
	-A ///etc/fstabに記載されているファイルシステムをすべてチェック
e2fsck [オプション] デバイスファイル名 //ext2, ext3, ext4ファイルシステムの不整合をチェックし必要であれば修正。これを実行するときはファイルシステムをアンマウントする。そうしないと問題のないファイルを削除してしまう可能性がある。
	-p //軽微なエラー(参照カウントの相違など)は尋ねることなく自動修正する。それ以外のエラーは修正せずにそのまま終了
	-a //-pと同じ動作をする。広報互換性のためのオプション。-pの使用が推奨されている。
	-n //fsckの質問に対して、すべてnoと答える。ファイルシステムを修正せず、どのようなエラーがあるか調べるときに使用
	-y //fsckの質問に対して、すべてyesと答える。ファイルシステムのエラーはすべて整合性を保つ操作によって修正される。その結果解いて不整合の原因となっているファイルが消されることがある。
	-r //検知したエラーに対してyes/noを質問することにより対話的に修正する。互換性のためのオプションであり、デフォルトの動作である。
debugs [オプション] [デバイスファイル名] //ext2, ext3, ext4ファイルシステムの対話的なデバッガツール。ファイルシステムの状態の調査・変更を行うことが可能
	-w //ファイルシステムを読み書き可能なモードでオープン
	-l コマンドファイル //コマンドファイルからのコマンドを読み込ませ、実行する
	stat filespec //filespecで指定したiノードの構造体の内容を表示する。filespecはiノード番号を<>で囲むか、パス名で指定する。
	dump [-p] file spec out_file //filespecで指定したiノードと、out_fileで指定したファイルを関連づける。-pオプションを与えると、out_fileの所有者(owner)、所有者グループ(group)、パーミッション(permission)をfilespecと同じにする。
	quit //debugfsを終了する
mount [オプション] [デバイスファイル名(ファイルシステム)] [マウントポイント] //事前に作成したディレクトリ(マウントポイント)に対し、ファイルシステムをマウント(接続)する。引数なしで現在のマウント情報を表示
	-a ///etc/fstabファイルに記載されているファイルシステムをすべてマウントする
	-f //ファイルシステムを読み取り専用でマウントする。 -o roと同意
	-w //ファイルシステムを読みかき可能なモードでマウントする(デフォルト)。 -o rwと同意
	-t //ファイルシステムタイプを指定してマウントする
	-o //マウントオプションを指定する
	-bind //サブツリーをどこか他の場所に再マウントする
lsblk //利用可能なブロックデバイスの一覧を表示
blkid //利用可能なブロックデバイスの情報を表示
unmount [オプション] マウントポイント|デバイスファイル名 //特定のファイルシステムをルートファイルシステムから切り離す
	-a ///etc/fstabファイルに記載されているファイルシステムをすべてアンマウント
	-r //アンマウントが失敗した場合、読み取り専用での再マウントを試みる
	-t //指定したタイプのファイルシステムのみに対してアンマウント
ps [オプション] //プロセスの情報を表示
	-e //すべてのプロセスを表示
	-f //階層表示
	-l //長いフォーマットで表示
	-r //現在実行中のプロセスのみ表示
source ファイル名 //ファイルに書かれたコマンドを現在のシェルで実行
id [オプション] [ユーザ名] //自分がログインしているユーザ、および所属グループを表示
	-u //実効ユーザのみをIDで表示
	-g //実効グループのみをIDで表示
	-G //ユーザが属するグループを全て表示
	-n //ユーザやグループの名前のみ表示
	-Z //セキュリティコンテキストを表示(引数なし)
groups [ユーザ名] //自分の所属グループを表示
chmod [オプション] モード ファイル名 //ファイルやディレクトリのパーミッションを変更する
	-R //ディレクトリに指定した場合、サブディレクトリも含め再帰的にパーミッション変更
　モード一覧
	u:所有者, g:グループ, o:その他, a:すべてのユーザ
	+:許可を与える, -:許可を削除, =:許可を設定
	r:読み取り, w:書き込み, x:実行権
chmod u+s ファイル名 //ファイルにSUIDを設定。u+sを(4000+現在のパーミッション)にしてもOK
      g+s ディレクトリ名 //ディレクトリにSGIDを設定。g+sは(2000+現在のパーミッション)でも可
      o+t ディレクトリ名 //ディレクトリにスティッキービットを設定。o+tは(1000+現在のパーミッション)でも可
複数ユーザで使用するディレクトリに対してSGIDを設定すると、誰がファイルを作成しても所有グループを同じにできる(1次グループはユーザー名と同じグループであるため)。
umask //現在設定されているumask値を確認。値を変更する場合、現在のプロセスと子プロセスのみ有効。初期設定とする場合シェルの設定ファイルを変更する必要あり。
chown [オプション] ユーザ名.[グループ名] ファイル名|ディレクトリ名 //指定されたファイルの所有者とグループを指定されたユーザとグループに変更(sudoが必要)
	-R //ディレクトリにのみ指定可能、サブディレクトリを含め再帰的にグループ変更
ln [オプション] オリジナルファイル名 リンク名 //オプションなしでハードリンクを作成
	-s //シンボリックリンクを作成
rm [オプション] ファイル名|ディレクトリ名 //指定したファイル、ディレクトリを削除
	-f //削除する際確認をしない
	-i //削除する際確認する
	-r //ディレクトリを再帰的に削除
ldconfig [オプション] ディレクトリ名|ライブラリ //ldconfigは/etc/ld.so.confファイルを参照して適切なシンボリックファイルを作成し、共有ライブラリのパスを/etc/ld.so.cacheに書き込む。
	-v //ライブラリのバージョンやリンクメッセージなどを表示
	-n ディレクトリ名 //指定したディレクトリを追加で処理する
	-f //ライブラリパスファイルを指定する
rpm [オプション] パッケージ //指定したパッケージの情報を表示
	-q //指定したパッケージがインストールされていればバージョンを表示
	-a //インストール済みのrpmパッケージ情報を一覧で表示
	-i //指定したパッケージの詳細情報を表示
	-f //指定したファイルを含むrpmパッケージを表示
	-c //指定したパッケージ内の設定ファイルのみを表示
	-l //指定したパッケージに含まれるすべてのファイルを表示
	-K //パッケージの完全性を確認するために指定されたパッケージファイルに含まれるすべてのダイジェスト値と署名をチェックする
	-R //指定したパッケージが依存しているrpmパッケージ名を表示
	-p //インストールされたrpmパッケージではなく、指定したrpmパッケージファイルの情報を表示
	-i //パッケージをインストール
	-U //パッケージをアップグレードする。インストール済みのパッケージが存在しない場合、新規にインストールを行う
	-F //パッケージをアップデートする。インストール済みのパッケージが存在しない場合、何も行わない
	-v //詳細な情報を表示する
	--force //指定されたパッケージがすでにインストールされていても上書きインストールを行う
	--oldpackage //古いパッケージに置き換えることを許可する
	--test //パッケージをインストールせず、衝突等のチェックを行い結果を表示する
yum サブコマンド //rpmパッケージを管理するユーティリティ。
	list //利用可能な全rpmパッケージを表示
	list installed //インストール済みのrpmパッケージを表示
	info //指定したrpmパッケージの詳細情報を表示
	search //指定したキーワードでrpmパッケージを検索し結果を表示
	deplist //指定したrpmパッケージの依存情報を表示
	list updates //インストール済みのrpmパッケージで更新可能なものを表示
	install //指定したrpmパッケージをインストールする。
yumdownloader パッケージ名 //パッケージをインストールせずにダウンロードする
zipper [グローバルオプション] コマンド [コマンドオプション] [パラメータ] //Libzyppライブラリを利用したコマンドラインによるパッケージ管理ツール

dpkg オプション パッケージ名 //debパッケージの管理
	-l //インストール済みの全debパッケージの情報を表示。特定のパッケージ情報のみ表示する場合は-lオプションの後にパッケージ名を指定
	-s //指定したパッケージの情報を表示。依存関係も確認可能
	-L //指定したパッケージに含まれるすべてのファイルを表示
	-S //インストール済みのパッケージから指定されたファイルを検索
	-I //指定したパッケージの詳細情報を表示
	-C //インストールが完了していないパッケージを検索
	-c //インストールされたパッケージではなく、指定したdebパッケージファイルに含まれるファイルの一覧を表示
